Поддерживается аудит следующих ОС семейства Unix:
- Canonical Ubuntu 16.04, 18.04, 20.04;
- CentOS 6.x, 7.x, 8.x;
- Debian 9 ,10;
- FreeBSD 11, 12;
- HPE HP-UX 11.31;
- IBM AIX 7.1, 7.2;
- Oracle Linux 6.x, 7.x, 8.x;
- Oracle Solaris 11.0-11.4;
- Red Hat Enterprise Linux 6.x, 7.x, 8.x;
- SUSE Linux Enterprise Server 15.

Настройку актива нужно выполнять от имени учетной записи root.

Внимание! При использовании в IT-инфраструктуре организации межсетевого экрана или других средств для контроля сетевого трафика требуется настроить в них правила, разрешающие трафик в обоих направлениях между узлом актива и узлом MP 10 Agent. Для проведения аудита актива по протоколу SSH используется порт TCP 22.

Внимание! На узле актива должна быть установлена программа sudo.

Для проведения аудита на активе нужно:
1) Создать учетную запись пользователя ОС для доступа к активу MP 10 Agent. В домашнем каталоге учетной записи создать каталог bin для файлов сценариев. 
Примечание. Данные этой учетной записи нужно указать при добавлении учетной записи в MaxPatrol SIEM.
2) Поместить в каталог bin файлы сценариев для проведения аудита. В домашний каталог созданной учетной записи поместить файл .bash_profile (или .profile) с параметрами переменной окружения $PATH.
Примечание. Файлы сценариев и файл .bash_profile (.profile) входят в комплект поставки.
3) Настроить программу sudo и предоставить учетной записи права на выполнение команд для запуска сценариев.

Чтобы создать и настроить учетную запись:
1) Создайте учетную запись и укажите для нее пароль:
	- если установлена ОС FreeBSD:
		adduser
		Login: <Логин>
		Shell: sh
		Home directory: /home/<Логин>
		Password: empty - no, random - no
		<Пароль>
	- если ОС Oracle Solaris:
		useradd -m -d /export/home/<Логин> <Логин>
		passwd <Логин>
		<Пароль>
	- если другая ОС:
		useradd -m -g users -s /bin/bash <Логин>
		passwd <Логин>
		<Пароль> 
2) Настройте права доступа к домашнему каталогу:
	chmod 700 /home/<Логин> 
3) Авторизуйтесь под именем созданной учетной записи:
	su - <Логин>
	<Пароль>
4) Создайте каталог /home/<Логин>/bin:
	mkdir ~/bin 
5) Настройте права доступа к каталогу:
	chmod 700 -R ~/bin
Учетная запись создана.

Чтобы подготовить сценарии для проведения аудита:
1) Перенесите архив bin.tar, входящий в комплект поставки, в домашний каталог созданной учетной записи. 
2) Перейдите в домашний каталог:
	cd /home/<Логин> 
3) Распакуйте архив:
	tar -xf bin.tar 
4) Удалите архив:
	rm bin.tar: 
5) Перенесите файл в домашний каталог учетной записи:
	- если используется командная оболочка Bash — .bash_profile
	- если командная оболочка Bourne shell (sh) — .profile
	Примечание. Вы можете просмотреть список каталогов в переменной окружения $PATH, выполнив команду echo $PATH. Название каталога /home/<Логин>/bin должно располагаться первым в списке на экране.
Сценарии для аудита подготовлены.

Чтобы настроить программу sudo:
1) Откройте для изменения файл /etc/sudoers:
	visudo
2) Добавьте в файл строки из файла Sudoers.txt с псевдонимами для сценариев.
3) Добавьте в файл строки из файла Sudoers.txt, предоставляющие учетной записи права на выполнение команд для запуска сценариев.
4) Если в файле отсутствует строка с директивой Defaults env_reset, добавьте ее:
	Defaults env_reset
5) Если в файле есть строка с директивой `Defaults env_keep`и она содержит переменную окружения PATH, удалите эту переменную из строки (или всю строку, если переменная единственная).
6) Сохраните изменения и закройте файл.
Программа sudo настроена.
